💻 app.js (ALTERADO)

✅ extractCnpjs() - Função reenscrita com 5 padrões:

  1. Padrão formatado: XX.XXX.XXX/XXXX-XX
     Regex: /\\b\\d{2}\\.\\d{3}\\.\\d{3}\\/\\d{4}-\\d{2}\\b/g

  2. Padrão com espaços/caracteres: XX XXX XXX XXXX XX
     Regex: /\\b\\d{2}[\\s\\-.]*\\d{3}[\\s\\-.]*\\d{3}[\\s\\/\\-.]*\\d{4}[\\s\\-.]*\\d{2}\\b/g

  3. Padrão sem formatação: 14 dígitos
     Regex: /\\b\\d{14}\\b/g

  4. CNPJ após "CNPJ:" ou "CNPJ ="
     Regex: /(?:cnpj|cnpj:|cnpj\\s*=|cnpj\\s*:)[\\s]*([\\d.\\-/\\s]+)/gi

  5. Padrão em contexto de rodapé/empresa
     Regex: /(?:empresa|cnpj|registr)\\s*(?:n[°º]|:)?\\s*([\\d.\\-/\\s]{14,20})/gi

  Cada match é validado com validateCnpj() e formatado.

✅ findOrganization() - Expandida com 4 estratégias:

  1. Schema.org / JSON-LD ("name": "...")
  2. Meta tags OpenGraph (property="og:title")
  3. Copyright/Footer (© ou "Todos os direitos reservados")
  4. Meta tags customizadas (razao_social, company)

✅ extractCnpjsAggressive() - Nova função para casos difíceis:

  - Busca em atributos data-cnpj, data-company, data-empresa
  - Busca dentro de <script> tags (JSON inline)
  - Valida cada match encontrado
  - Retorna Set de CNPJs únicos

✅ fetchClueForPath() - Melhorada:

  - Agora chama extractCnpjsAggressive() se scraping simples não acha nada
  - Fallback inteligente (não tenta novamente se achou na primeira)
